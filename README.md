# SkyTrace（仮称）

> 世界中に散らばるUAP／UFO目撃情報、公式資料、報道、航空・衛星・天文・気象データを統合し、AIが「何が分かっていて、何がまだ分からないか」を出典付きで整理する、iPhone向け観測ニュースアプリ。

## 0. このREADMEの目的

この文書は、Claude Codeがプロジェクトの思想を壊さず、モックからApp Store提出可能な完成形まで自律的に実装するための入口である。

詳細は以下を正本とする。

- `docs/ARCHITECTURE.md`：プロダクト、UX、データ、AI、バックエンド、セキュリティ、App Store設計
- `docs/CLAUDE_CODE_PHASES.md`：Claude Codeが段階的に実装するための手順、完了条件、停止条件
- `CLAUDE.md`：Claude Codeが毎回守る常設ルール

## 1. プロダクトの核

SkyTraceは「宇宙人が来た」と断定するアプリではない。

本質は、同じ空の出来事について散在する情報を集め、次を明確に分けて提示することである。

1. 報告そのものはどれほど確かか
2. 証拠はどれほど良質か
3. 複数報告は本当に独立しているか
4. 航空機、人工衛星、流星、天体、気象、ドローン等でどこまで説明できるか
5. 現時点で残る未解明部分は何か
6. 後日、新情報によって評価がどう変わったか

表示原則：

- 「UFOが現れた」ではなく「UAPらしき現象が報告された」
- 「説明できない」ことを「地球外起源」と同一視しない
- AIの推測より、出典・証拠・不確実性を優先する
- 信じさせることと否定することの両方を目的にしない
- 科学的な慎重さと未知への好奇心を両立する

中心コピー：

> 世界の空に残された、まだ名前のないものを丁寧に記録する。

## 2. 想定ユーザー

### 一般ユーザー

- 今日、世界のどこで何が報告されたか知りたい
- UFO／UAPに興味はあるが、煽りや陰謀論だけでは物足りない
- 地図、短い記事、通知で気軽に追いたい

### 深く読みたいユーザー

- 複数報道の一致点・矛盾点を知りたい
- 航空・衛星・流星照合まで含めて読みたい
- 後日判明した説明や更新履歴を追いたい

### 研究・調査志向ユーザー（将来）

- 地域、時刻、形状、継続時間、説明候補などで絞り込みたい
- CSV／JSON等で分析したい
- 事例同士の類似性や時系列傾向を見たい

## 6. スコア設計

単一の「信ぴょう性」だけを出さない。

### Evidence Quality / 証拠品質

- 元映像の有無
- 撮影時間・継続時間
- メタデータ
- 解像度
- 編集痕跡
- 方角・位置情報

### Independent Corroboration / 独立報告性

- 別人物か
- 別地点か
- コピー投稿ではないか
- 時刻・方角・特徴が一致するか

### Known Phenomena Match / 既知現象一致度

- 航空機
- 人工衛星
- ロケット／再突入
- 流星・火球
- 惑星・恒星
- 気象・雷・雲
- サーチライト
- ドローン
- カメラ由来の現象

この値は高いほど「既知現象で説明できる可能性が高い」。

### Unresolvedness / 未解明度

- 既存説明で残る差分
- 欠損データの多さ
- 矛盾の有無
- 異常運動等が客観的に確認できるか

未解明度が高いことは地球外起源を意味しない。

### 表示状態

- `explained`：既知現象との一致が強い
- `likely_explained`：説明候補が優勢
- `insufficient_data`：情報不足
- `under_review`：調査中
- `notable_unresolved`：比較的良質な証拠があり未解明部分が残る
- `disputed`：情報間の重大な矛盾
- `withdrawn`：誤報・転載・投稿撤回等

## 7. 技術方針

### iOS

- Swift 6
- SwiftUI
- iOS 17以上を最低対応候補
- Xcode 26以上、iOS 26 SDK以上で提出
- Observation
- async/await
- MapKit
- SwiftData（キャッシュ、ブックマーク、設定）
- StoreKit 2
- UserNotifications
- AuthenticationServices（アカウント追加時）
- URLSessionベースのAPIクライアント
- XCTest / XCUITest

iOS 26の表現は段階的に採用し、iOS 17〜25では自然にフォールバックさせる。

### Backend

- Python 3.12+
- FastAPI
- PostgreSQL + PostGIS
- SQLAlchemy 2
- Alembic
- Redis + 軽量ジョブキュー
- オブジェクトストレージ（S3互換）
- Docker / Docker Compose
- OpenAPI
- pytest

### Admin

- Next.js
- TypeScript
- App Router
- 認証必須
- 事例統合、記事レビュー、公開、訂正、出典管理、ジョブ監視

### AI

- サーバー側のみ
- LLMプロバイダーは抽象化
- 構造化JSON出力
- 文章の各主張を証拠IDへ紐付ける
- 推測は推測と明示
- 引用不能な文を公開しない
- 高注目事例は人間レビューを必須にできる

APIキー、DBキー、AIキーをiOSアプリへ埋め込まない。

## 8. データ取得原則

### 優先順位

1. 政府・公的機関・科学機関
2. 信頼できる報道機関
3. 利用許諾を得た専門データベース
4. オープンデータ
5. SNS・動画投稿等は「手掛かり」として扱う

### 禁止

- 規約で禁止された自動取得
- 有料記事の本文再配布
- 無断の画像・動画ホスティング
- 出典を隠したAI要約
- 著作権者の文章を長く再掲載

NUFORC、MUFON等の専門データベースは、利用条件を確認し、必要に応じて書面許可または正式提携を得るまで本番自動取得しない。

外部情報源は必ずProvider Adapterとして実装し、利用許可がない場合は無効化できるようにする。

## 9. AI記事生成の原則

生成パイプライン：

1. 収集
2. 重複除去
3. 日時・場所・単位の正規化
4. 同一事象候補のクラスタリング
5. 主張抽出
6. 出典と証拠の紐付け
7. 既知現象との機械照合
8. 一致点・矛盾点・不足情報の生成
9. 構造化記事ドラフト
10. 引用整合性検査
11. 誇張・断定・名誉毀損・個人情報検査
12. 自動公開または人間レビュー
13. 後日更新と版管理

公開可能な各文は、次のいずれかでなければならない。

- `fact`：出典に直接支えられる
- `calculated`：計算・照合結果で、方法と入力を追跡できる
- `inference`：根拠付き推論として明示
- `unknown`：不明であることの記述

「根拠IDがない断定文」は公開しない。

## 10. App Store方針

- 提出ビルド：Xcode 26以上、iOS 26 SDK以上
- 最低対応OS：原則iOS 17以上
- デジタル購読：StoreKit 2
- 購読画面：価格、期間、自動更新、無料体験条件、利用規約、プライバシー、復元を明示
- すべてのリンクを有効化
- サポートURLとプライバシーポリシーURLを用意
- App Privacyの申告内容と実装を一致させる
- Privacy Manifestと利用SDKの要件を確認
- 通知、写真、位置情報等は必要な場面でのみ許可要求
- 初回起動直後に通知や位置情報を要求しない
- アカウント作成を導入した場合、アプリ内から削除開始可能にする
- 第三者ログインを導入する場合はSign in with Apple要件を満たす
- 初回リリースはUGC非搭載
- App Review Notesに、AI記事、スコアの意味、購読テスト方法、デモデータの有無を明記

## 13. Definition of Done

「完成」と呼べる最低条件：

- iPhone実機またはSimulatorで主要画面が動く
- オフライン時も最後に取得したデータを読める
- APIが落ちてもクラッシュしない
- 全事例に出典と取得日時がある
- AI記事の各主張に根拠がある
- Plus購読、復元、失効、猶予期間を扱える
- VoiceOver、Dynamic Type、Reduce Motionに対応
- Privacy ManifestとApp Privacy申告用のデータ一覧がある
- プライバシー、利用規約、サポートページが実在する
- 単体・統合・UIテストが通る
- TestFlightで実機確認済み
- App Store用スクリーンショット、説明文、レビュー向け手順が揃う
- 外部情報源の権利状態が一覧化され、未許可ソースは本番無効

## 14. 開発上の優先順位

優先度は常に以下の順。

1. 出典と検証可能性
2. App Store審査適合
3. クラッシュしないこと
4. 読みやすさと操作性
5. データ鮮度
6. AI文章の美しさ
7. 演出

宇宙的な美しさは重要だが、証拠の見え方を邪魔してはならない。
